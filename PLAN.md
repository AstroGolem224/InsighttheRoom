# Plan: Insight the Room (ItR) — Android room-scan → floorplan app
_Locked via grill — by Claude (Forge) + Matthias. Revised after Codex Rounds 1–5 (MAX_ROUNDS)._

## Goal
An Android app that lets a user scan a single room live with the phone camera (ARCore), producing a 2D floorplan with real-world dimensions and the positions of detected furniture on that plan. Everything runs on-device (no cloud). v1 is one room per scan; the data model uses a room-local coordinate frame so full building stitching can be added later with minimal (not zero) rework. Optimization suggestions and photo/video import are explicitly deferred to v2+.

## Approach
1. **Tech stack:** Kotlin, Jetpack Compose UI, Hilt DI, ARCore via [SceneView-Android](https://github.com/SceneView/sceneview-android) (Apache-2.0, Filament rendering), MediaPipe LiteRT Object Detector (Apache-2.0), Room (SQLite) for persistence. Multi-module Gradle. min SDK 26, ARCore-required (Play-gated to supported devices). All dependency versions pinned before implementation (see Dependency pinning).
2. **ARCore lifecycle (core-arcore):** explicit state machine covering ARCore availability → install/update → camera-permission grant/deny → session create → pause/resume → tracking loss/recovery → error. Plane detection = horizontal **and** vertical, enabled on **all** supported devices. Wall pre-suggestion derives from vertical-plane ∩ floor intersections (no Depth needed). Depth (`Config.DepthMode`) stays **disabled** in v1 (no feature uses it); reserved for future footprint/auto-wall work.
3. **Floor selection + scan (feature-scan):** ARCore may track many horizontal planes (tables, etc.), so the user explicitly selects/confirms **one floor plane** first. All corner taps are constrained to hits on that plane within a documented tolerance; off-plane hits are rejected with feedback.
4. **Corner capture → room-local frame:** user taps room corners in order via an AR reticle on the floor plane. Every tapped 3D point is **projected onto the fitted floor plane** before use (kills height noise from plane refinement). A **room-local orthonormal basis** is defined explicitly: Y = normalized floor-plane normal (up), X = first valid wall edge projected onto the floor plane, Z = X×Y; origin = first corner. This fixes yaw (a normal+origin alone leaves rotation undefined). All geometry is transformed into it and persisted in room-local coordinates. **The basis is locked once the floor and first wall edge are confirmed — before object detection starts** — so tracks/markers never build on a shifting frame. If a basis-defining corner is later edited, every corner, snap, object track, and marker is **rebased** as one unit: every frame/track is stamped with a **basis revision**, inference is **paused and drained** before the rebase, and any late detection callback from an older revision is discarded or explicitly transformed (a DB transaction alone can't cover in-flight in-memory tracks/callbacks). **No ARCore anchors or session-world coordinates are persisted** (session-relative, non-durable). **Floor plane — two separated responsibilities:** (a) *hit eligibility* follows the **live** subsumption chain (`getSubsumedBy()`) so taps validate against the current trackable polygon; (b) *metric projection* uses **one immutable reference plane** snapshotted at confirmation. The reference plane never silently changes (that would rotate/translate the whole room-local frame); changing it requires explicit user recalibration. **Hybrid walls:** app pre-suggests corners/walls from **vertical-plane ∩ floor-plane intersections** (needs only standard plane detection, not Depth) with a confidence threshold; user confirms/corrects. Manual corner-tapping is always the reliable base. (Depth API is not required by any v1 feature; kept optional for future use.)
5. **Geometry (pure-Kotlin, no Android dep):** projected corners → polygon validation (min vertices, min edge length, self-intersection, collinearity, winding, closure) with actionable correction UI → wall segments → lengths + shoelace area. **Manhattan-snap is non-destructive and never overwrites canonical data:** persistence stores raw ordered corners (canonical) **plus a separate optional snap transform / snapped-corner set**. Raw walls and displayed (snapped) walls are derived independently; the snap is a previewable/reversible overlay showing the adjustment delta. Correctness-critical core, isolated for emulator-free JVM unit tests.
6. **Ceiling height:** explicit operation — two-point measurement (floor tap + ceiling tap); height = displacement between the two points **projected onto the floor-plane normal** (not raw Euclidean distance, which is wrong when taps are horizontally offset). Reject implausible/weakly-tracked results; numeric entry as alternative. No silent default committed as measured; if unmeasured, flagged as assumption.
7. **Object detection (in feature-scan, shares the frame pipeline):** a **single ARCore-frame acquisition pipeline** owns the camera (SceneView owns ARCore; MediaPipe is fed from ARCore images, **not** a separate CameraX). Bounded in-flight work, explicit backpressure, dropped-frame handling. **Ownership contract:** frame pixels are **copied** for MediaPipe before `Image.close()` (MediaPipe must not retain the backing ARCore image); each frame record stays alive until its result, rejection, timeout, or detector shutdown. Each submitted frame is stored with **pose, intrinsics, display transform, crop/rotation, timestamp**; MediaPipe results (which arrive late) are projected only against their exact source-frame record — never the current frame. Placement ray uses the detection box **bottom-center**, constrained to the floor plane; non-floor intersections are rejected. Detections are **tracked across frames** by a conservative combination of class + image-space IoU + 3D room-space position + observation history (proximity alone would merge two adjacent identical chairs); user gets explicit split/merge controls. Persisted only after user confirmation. **Marker editing is a first-class scan interaction:** reposition (drag on floorplan), delete, relabel. COCO default classes (chair, couch, bed, dining table, tv, potted plant, refrigerator, …). Placement is a labeled marker point (no footprint volume in v1); low-confidence/tall-item (TV) placements are flagged.
8. **Floorplan render — single display list (core, pure Kotlin):** one platform-neutral **floorplan display list** (polygon, per-wall dimensions, area label, furniture markers, scale) is generated once and consumed by all three renderers — Compose Canvas (interactive), SVG, PNG. Guarantees **geometry/style** parity via a shared `RenderTransform` (quantized coordinates) + shared `RenderStyle` (colors/sizes/offsets) consumed identically by all three renderers, plus a shared generic font family (`sans-serif`). **v1 scope (amended 2026-07-13):** pixel-perfect cross-renderer TEXT parity — a single embedded/bundled font file + per-renderer text-measurement adapters + text golden tests — is **deferred to v2** (it needs screenshot infrastructure and only affects label sub-pixel placement, not geometry). v1 acceptance = geometry/style invariants from the one display list.
9. **Persistence (persistence):** Room with **normalized entities** — `BuildingEntity`, `RoomEntity`, ordered `CornerEntity` (explicit index), `RoomObjectEntity`, plus optional ordered `SnappedCornerEntity(roomId, index, x, z)` (unique `(roomId, index)`, cascade-delete with the room) — foreign keys with cascades, stable IDs, transactional aggregate save, schema version + migrations, draft-scan state. **Draft recovery after process death = view / export / restart only, not resume live capture** — a fresh ARCore session can't relocalize to an unfinished draft (no anchors/session-world coords persisted); on-device relocalization is out of scope for v1. **Exporting an invalid/unclosed draft** either is disabled or produces an explicitly **watermarked partial plan that omits area and any unsupported dimensions** (never a bogus measured value). **Ordered raw corners are the sole canonical geometry; walls (raw and snapped) are always derived, never stored as an editable second source of truth.** Snap is persisted as those ordered per-vertex `SnappedCornerEntity` rows (Manhattan orthogonalization is not one global transform) alongside the raw corners, never overwriting them; preview and export render the snapped corners, the disclaimer/delta references the raw. Domain names disambiguate from Android `Room` (`ScannedRoom`/`RoomEntity`). **Cloud Auto Backup disabled** (both legacy backup rules and `dataExtractionRules`) so "on-device only" holds.
10. **Export:** split to honor the pure-Kotlin boundary. `export-core` (pure Kotlin) consumes the shared display list → **SVG** with all text/attributes XML-escaped and size limits. `export-android` rasterizes to **PNG** (Canvas→Bitmap) and shares both via **FileProvider** from a scoped cache subdirectory with read-only **intent-scoped** grants (preferred over direct package grants). Cleanup uses a generous cache TTL + next-launch sweep (not immediate deletion, which would race a recipient that opens the URI late); any directly-issued grants are revoked only after a safe lifecycle boundary.

## Module map (start slim, split later)
| Module | Purpose | Android dep? |
|--------|---------|--------------|
| `app` | entry, navigation, Hilt setup, ARCore lifecycle host | yes |
| `core` | domain types (`Building`/`ScannedRoom`/`Wall`/`Corner`/`RoomObject`/`FloorPlan`) **+ geometry** (validation, walls, dimensions, area, snap, room-local transform incl. yaw) **+ floorplan display-list builder** | **no (pure Kotlin, JVM-tested)** |
| `core-arcore` | ARCore session + lifecycle state machine, plane/depth capture, single-frame pipeline, hit-tests, frame records | yes (ARCore, SceneView) |
| `feature-scan` | floor selection, corner-tap flow, guidance, **MediaPipe detection + tracking** (shares frame pipeline) | yes (Compose, MediaPipe) |
| `floorplan` | 2D Canvas render, dimensioning, object markers | yes (Compose Canvas) |
| `persistence` | normalized Room schema, transactional save/load, migrations | yes (Room) |
| `export-core` | render-commands + SVG (escaped) | **no (pure Kotlin)** |
| `export-android` | PNG rasterization + FileProvider sharing | yes |

Rationale for slimming: `detection` and `core-arcore`/`feature-scan` must share timestamped frame state; separate modules invite cycles/leaky abstractions. `geometry` folds into `core` but stays pure-Kotlin/JVM-tested. Split further only once interfaces stabilize and independent build/test value emerges.

## Key decisions & tradeoffs
- **Live AR scan, not photo import (v1).** ARCore needs live motion for metric scale; import would need a photogrammetry backend and yields only estimated scale. v2.
- **Hybrid walls, manual base; Depth disabled in v1.** Wall pre-suggestion = vertical-plane ∩ floor intersection, runs on all devices, needs no Depth. Depth mode stays **off** throughout v1 (avoids the processing cost of an unused feature); device benchmarking is by performance tier, not Depth capability. Manual corner-tapping is the deterministic always-available base.
- **Room-local coordinate frame, no persisted anchors.** ARCore world/anchor coords are session-relative; a room-local orthonormal frame (floor normal + origin corner) is the durable coordinate system. Also the seam for future multi-room stitching (add per-room `roomToBuilding` transform + door/opening entities).
- **One room per scan (v1).** Building stitching (end goal) needs cross-room drift correction, door anchors, adjacency — a separate hard problem. Room-local frames now → minimal (not zero) rework later; the "no rewrite" claim is retracted.
- **Bottom-center + floor-constrained object placement, tracked & confirmed.** Box-center rays land behind objects; bottom-center on the floor plane, cross-frame tracking, and user confirmation prevent bad/duplicate markers. Footprint rectangles = v2.
- **Non-destructive Manhattan snap.** Raw measurements immutable; snap is a reversible overlay with a visible error delta — cosmetic cleanup never silently corrupts data.
- **PNG + SVG, boundary-correct.** Pure `export-core` (SVG/render-commands) + `export-android` (Bitmap/PNG) resolves the pure-Kotlin contradiction. No CAD/IFC in v1.
- **MediaPipe over Ultralytics YOLO.** YOLO weights are AGPL-3.0; MediaPipe/EfficientDet-Lite is Apache-2.0 with COCO furniture classes, zero training.
- **ARCore-required, Depth optional.** Gate install to ARCore devices (min SDK 26); full runtime lifecycle handling (availability/install/permission/pause/tracking-loss/draft-recovery), not just Play gating.

## Phase 0 — compatibility spike (mandatory exit gate before feature work)
De-risks the young/churning stack. Exit criteria, all required:
- Exact pinned versions committed (Gradle dependency verification): Kotlin, Compose BOM, Hilt, SceneView-Android, ARCore, Filament (transitive via SceneView — verify match), MediaPipe Tasks Vision, Room.
- Exact MediaPipe `.tflite` artifact + **SHA-256** pinned; dataset/license/class allowlist documented.
- A compiling prototype proving the critical integration: ARCore frame → copied buffer → MediaPipe detection → bottom-center ray → floor-plane hit, on-device.
- Benchmark evidence: CPU vs GPU delegate latency + score threshold on **≥3 devices spanning high-end, mid-range, and a low-end supported (API-26, ~2–3 GB RAM) device** — or an explicit, documented exclusion of devices below a stated RAM/GPU tier in the Play listing.
No feature module starts until this gate passes.

## Acceptance criteria (v1 release gate) — concrete
Measurement protocol: 60 s sustained scan, after 10 s warm-up, reported at p50/p95, at normal thermal state (note if throttled), **plus a longer max-duration soak test** to expose scan-length leaks and thermal degradation. Dropped frames counted, not hidden.
- **Dimension accuracy:** wall length error ≤ 3% or ≤ 5 cm (whichever larger) vs tape-measured ground truth; area error ≤ 5%. Rooms up to 8×8 m, normal indoor lighting.
- **Object placement (auto and corrected gated separately):** accuracy measured against the object's **visible floor-contact point** (bottom-center is a contact point, not the object center). Report **automatic** placement error, **final** (post-reposition) error ≤ 30 cm, correction rate, and rejected-placement rate — so the gate can't be passed by manual correction masking broken auto-placement. Tall/wall items (TV) flagged, not error-budgeted.
- **Performance (pass/fail):** render frame time **p95 ≤ 50 ms** (≥20 FPS); **delivered** detection rate **≥ 4 Hz** with inference p95 ≤ 80 ms; **dropped-frame rate ≤ 15%**; total **PSS (incl. native + GPU)** under an **empirically-derived per-device-tier limit** (`memoryClass` is only the managed-heap budget, not a PSS ceiling — don't equate them); zero low-memory kills over the sustained scan and the soak test.
- **No-cloud enforcement:** app declares **no `INTERNET` permission**; merged manifest + bundled-SDK audit; a release-build network-traffic test confirms **zero egress for the app UID** during scan+export (measured after ARCore/Play-Services prerequisites are installed, so their install/update traffic isn't attributed to the app).
- **Tests:** (a) pure-Kotlin geometry unit tests — validation, walls, area, snap, room-local transform incl. yaw; (b) recorded-session integration tests replaying captured ARCore frames, asserting **tolerant invariants** (poses/timing vary across playback runs) not exact poses; (c) Room migration tests + interrupted-transaction save; (d) renderer golden tests (geometry/style invariants from one display list); (e) malicious-label export/escaping tests; (f) lifecycle state-machine tests incl. permission denial + process-death draft recovery.

## Observability (lightweight, opt-in)
Local-only diagnostic log (opt-in): pose/tracking-quality state, plane revisions, dropped inference frames, rejected hit reasons, model metadata. Plus a **small user-triggered sanitized export** (timings, tracking/plane/rejection events, model metadata only — **excludes camera images and raw room geometry**, with retention/size limits) so field issues are diagnosable. No network. Full ARCore-recording debug bundle deferred to v2.

## Risks / open questions
- **ARCore tracking drift / scale accuracy:** single-room scope, tracking-quality indicator, re-tap; accuracy is "sketch/estimate", not survey-grade — surfaced disclaimer + measured error budget above.
- **Occluded corners** (furniture against wall): accept a wall-plane hit only to compute the **wall-plane ∩ floor-plane intersection line**, then let the user pick the corner point on that line — keeps the floor-only constraint intact (no raw wall points enter geometry).
- **Recorded-session test fixtures** contain camera video + sensor + device fingerprint. Capture only in **staged, non-sensitive rooms** (no post-hoc frame blurring — that would alter ML detections and playback); protect the originals in access-controlled test storage, version+hash them, and exclude from the repo (`.gitignore`, never committed).
- **Ceiling plane may not track:** two-point measure or numeric entry, flagged if assumed.
- **SceneView-Android version churn:** young library; pinned + dependency verification, watch breaking changes.
- **UI/UX not yet designed** — deferred to the next step per the user; this plan covers architecture, deliverables, and integration contracts only.

## UI / UX (v1)
Material 3 (Material You dynamic color), light + dark, technical-clean floorplan (thin wall lines, cm/mm dimensions, furniture as labeled points). **4 top-level screens:**
1. **Home / list** — saved scans as cards (floorplan thumbnail, area, object count); FAB "Neuer Scan". One `Building` = one room in v1.
2. **Scan wizard** — guided, one goal per stage, mapping 1:1 to the ARCore state machine: (1) floor select + confirm → (2) tap corners in order (live edge length, undo) → (3) ceiling height (two-point or numeric) → (4) furniture scan (detect + confirm/reposition/relabel markers) → (5) review. Wizard structure minimizes AR error states; expert/freeform mode = v2.
3. **Floorplan detail** — finished plan: polygon, per-wall dimensions, area, furniture markers; edit markers (drag/delete/relabel), export (PNG/SVG via FileProvider share).
4. **Settings** — units (metric/imperial), Manhattan-snap default toggle, opt-in diagnostic log, sanitized-export trigger, "About" + accuracy disclaimer.

Themes beyond default Material 3 (e.g. blueprint skin), onboarding tutorial = v2.

## Out of scope (v1)
Photo/video import & photogrammetry (COLMAP/RoomFormer); multi-room building stitching; optimization/redesign suggestions; auto panorama→floorplan (HorizonNet); footprint/3D object boxes; CAD/BIM export (PDF/glTF/IFC); iOS; cloud/backend/accounts/sync; custom-trained classes beyond MediaPipe COCO; **full ARCore-recording debug bundle** (the small sanitized diagnostic export stays in v1 scope).
